## Anticipated Scope (Informative, Subject to Change)

I4C is anticipated to be a clean-slate logical protocol that:

- Shares I3C's physical layer (two-wire SDA/SCL, same electrical levels,
  same timing envelope).
- Supports dual-mode coexistence with legacy I3C on the same bus.
- Redesigns the logical protocol to make timestamping, Byzantine fusion,
  QoS, and provenance first-class primitives rather than layered
  extensions.
- Provides an efficient wire format without the envelope overhead that
  I3C-EX incurs.
- Targets FPGA validation (Digilent Cora Z7 / Terasic DE10-Nano) per the
  MDPI open-source I3C controller precedent.

**None of the above is binding.** The real design follows Paper 1.
